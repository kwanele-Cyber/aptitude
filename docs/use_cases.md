# 📋 Aptitude Use Cases

This document serves as the master list of use cases for the Aptitude platform.

## 👤 User Use Cases

| ID | Use Case Name | Description |
| :--- | :--- | :--- |
| **U01** | **User Registration & Profile Creation** | Users sign up and create profiles with the skills they offer and want to learn, including availability and location. |
| **U02** | **Skill Posting (Offer Skills)** | Users post skills they can teach, including descriptions, levels, and preferred exchange skills. |
| **U03** | **Skill Request Posting** | Users request to learn specific skills, including schedule and format preferences. |
| **U04** | **AI Matchmaking** | The system matches users with complementary skills, availability, and proximity using AI. |
| **U05** | **Skill Exchange Agreement** | Two users agree on terms (time, format, frequency) for exchanging skills. |
| **U06** | **Chat & Communication** | Users communicate through in-app messaging to coordinate and discuss exchanges. |
| **U07** | **Session Scheduling** | Users schedule sessions with reminders and calendar support. |
| **U08** | **Conduct Skill Exchange Session** | Users meet (online or offline) to exchange skills. |
| **U09** | **Rating & Feedback** | Users rate and review each other after completing sessions. |
| **U10** | **AI Trust & Reputation Scoring** | AI calculates trust scores based on user behaviour, ratings, and activity. |
| **U11** | **Skill Discovery & Search** | Users search or browse for skills and other users based on interests and location. |
| **U12** | **Notifications System** | Users receive alerts for matches, messages, session reminders, and feedback. |
| **U13** | **Dispute Handling** | Users report issues, and the system/admin resolves conflicts. |
| **U14** | **Skill Progress Tracking** | Users track completed sessions and skills learned over time. |
| **U15** | **AI Skill Recommendations** | AI suggests skills to learn or teach based on user activity and trends. |
| **U16** | **Trust-Based Match Filtering** | Users filter matches based on reputation, verification, and trust scores. |
| **U17** | **Multi-Format Learning Options** | Users choose between online, in-person, or hybrid learning formats. |
| **U18** | **Admin Monitoring & Moderation** | Admins monitor activity, manage users, and handle reports with AI assistance. |
| **U19** | **View User Profile** | Users can view other users’ profiles, including skills and ratings. |
| **U20** | **Match Interaction** | Users can accept, reject, or ignore match suggestions. |
| **U21** | **Manage Skills** | Users can edit or delete skill offers and requests. |
| **U22** | **View Session History** | Users can see past and upcoming sessions. |
| **U23** | **Filter Matches** | Users can filter matches based on preferences. |
| **U24** | **View Platform Rules & Policy** | Users can view and read platform policy and rules. |
| **U34** | **Geo-Proximity Scoring** | System calculates Haversine distance with 5-min GPS updates for enhanced matching. |

## 🛡️ Admin Use Cases

| ID | Use Case Name | Description |
| :--- | :--- | :--- |
| **A01** | **Admin Login** | Admin securely logs into the system. |
| **A02** | **View Dashboard** | Admin views system overview including users, sessions, and reports. |
| **A03** | **Manage Users** | Admin can view, edit, suspend, or delete user accounts. |
| **A04** | **Handle Complaints** | Admin reviews reported issues and takes appropriate action. |
| **A05** | **Moderate Content** | Admin removes inappropriate or misleading content. |
| **A06** | **Moderate Skills** | Admin edits or removes skill listings that violate guidelines. |
| **A07** | **Manage Matches** | Admin views and intervenes in problematic matches. |
| **A08** | **Oversee Sessions** | Admin monitors sessions for irregularities or disputes. |
| **A09** | **Enforce Penalties** | Admin applies warnings, suspensions, or bans (e.g., 3-strike rule) to users who violated platform rules. |
| **A10** | **Manage Ratings & Feedback** | Admin reviews and removes abusive or false feedback. |
| **A11** | **Send Notifications** | Admin sends system-wide or targeted notifications. |
| **A12** | **Manage Disputes** | Admin resolves disputes between users and documents outcomes. |
| **A13** | **Support User Requests** | Admin assists users with system-related issues and queries. |
